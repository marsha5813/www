Jan. 14, 2017. Getting the most out of Stata's "capture" command.

If you use Stata regularly at home and work, you probably have loads of Stata .do files in a working directory that gets synced across multiple computers. In my case, that's a home desktop, a work desktop, and a laptop. The problem is that most of those .do files make calls to various files in different directories, e.g., C:\Me\Mystuff\Myproject\Myfile." That filepath is almost certainly different across the various computers you're using. Of course, you should be using relative instead of absolute file paths. Nevertheless, you may want to give Stata three of four file paths and ask it to just "use whichever one works." Ideally, Stata would try each file path, disregard the ones that don't work (i.e., that point to directories that don't exist on that particular computer), and apply the one file path that works. Enter the "capture" command. Watch the video for a quick tutorial.
